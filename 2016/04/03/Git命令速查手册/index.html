Git命令速查手册

发表于 2016-04-03   |   分类于 学习笔记   |  

早就想要系统的学习一下Git,今天终于开始行动了,对我这种懒癌晚期来说真是不容易啊!

想要系统的学习Git还请移步廖雪峰的Git教程,本文可以看成是这篇教程的速查版.

这个testGit项目是我在学习Git时的测试项目,期间尝试更换远程库时遇到了一些问题请见附:更换远程库流程及遇到的问题

创建版本库

初始化一个Git仓库: 

git init

添加文件到Git仓库,分两步:

git add <file1> <file2>
git commit -m <text>

时光机穿梭

查询工作区状态:

git status

查询项目修改内容:

git diff

版本回退

查看版本历史/单行显示

git log
git log --pretty=oneline

HEAD代表当前版本,版本穿梭:

git reset --hard <commit_id>

查看命令历史:

git reflog

工作区和暂存区

git add命令将要提交的修改放到暂存区,git commit将暂存区的修改提交到分支.

管理修改

Git不是跟踪文件,而是跟踪修改.

每次修改如果不add到暂存区,则不会被commit.

查看工作区和版本库里面最新版本区别:

git diff HEAD -- <file>

撤销修改

撤销工作区内容至最近一次git commit或git add:

git checkout -- <file>

撤销暂存区内容到工作区:

git reset HEAD <file>

撤销提交到版本库(未推送到远程库)的内容,请参考’版本回退’一节.

删除文件

删除版本库中的文件:

git rm <file>

将版本库中的文件恢复到工作区:

git checkout -- <file>

远程仓库

在本地生成SSH Key:

ssh-keygen -t rsa -C "youremail@example.com"

并在远程库设置SSH key.

添加远程库

关联本地库与远程库(默认为origin):

git remote add origin git@gitlab.com:cjlaaa/test.git

解除与远程库的关联(更换远程库时使用):

git remote rm origin

第一次将本地库master分支推送到远程库并关联(远程库为新建的空库时使用-u):

git push -u origin master

本地提交推送:

git push origin master

从远程库克隆

从远程库克隆:

git clone <website>

使用https速度较慢,稳定性差,每次都需要输入口令.(这里特指命令行,很多第三方客户端一般不需要每次输入)

使用ssh速度快,但需要设置ssh key.

分支管理

创建与合并分支

查看分支:

git branch

创建分支:

git branch <name>

切换分支:

git checkout <name>

穿件+切换分支:

git checkout -b <name>

合并某分支到当前分支(默认Fast-forward模式):

git merge <name>

删除分支:

git branch -d <name>

解决冲突

合并时如果出现冲突,只需要手动解决冲突,然后再次add并commit即可.

查看分支合并情况:

git log --graph    
git log --graph --pretty=oneline --abbrev-commit

分支管理策略

在团队开发中应该多使用分支功能.

合并时使用no-ff模式会保留分支信息:

git merge --no-ff -m <text> <name>

Bug分支

修复bug时最好创建新分支修复,然后合并并删除bug分支.

将未完成的工作储藏并清空工作区:

git stash

查看已经储藏的工作列表:

git stash list

恢复已经储藏的工作并删除储藏:

git stash apply
git stash drop

或:

git stash pop

恢复指定的已储藏工作:

git stash apply stash@{0}

Feature分支

开发一个新feature,最好新建一个分支.

强行删除没有合并的分支:

git branch -D <name>

多人协作

查看远程库信息:

git remote
git remote -v

本地新建的分支如果不推送到远程,则其他人不可见,可用来专门处理不需要与他人合作的工作,如修复bug.

在本地创建和远程分支对应的分支:

git checkout -b <branch-name> origin/<branch-name>

多人协作的工作流程:

  1. 推送自己修改:

    git push origin <branch-name>

  2. 如果推送失败,尝试合并远程分支到本地:

    git pull

  3. 如果遇到no tracking information错误,说明需要链接本地分支与远程分支:

    git branch --set-upstream <branch-name> origin/<branch-name>

  4. 如果有冲突则在本地解决冲突

  5. 再次尝试推送:

    git push origin <branch-name>

标签管理

标签相当于版本库的快照.

创建标签

创建标签(默认为HEAD):

git tag <name>

为某次提交创建标签:

git tag <name> <commit-id>

创建带信息的标签:

git tag -a <name> -m <text> <commit-id>

查看所有标签:

git tag

查看某标签详细信息:

git show <tag-name>

指定标签信息:

git tag -a <name> -m <text>

用PGP签名标签:

git tag -s <name> -m <text>

操作标签

删除标签:

git tag -d <name>

推送某一个标签到远程:

git push origin <tag-name>

一次性推送所有标签:

git push origin --tags

删除远程标签:

git tag -d <tag-name>
git push origin :refs/tags/<tag-name>

使用Github

  1. Fork他人的开源库.
  2. 在Fork过来的库上修改并上传.
  3. 推送pull request给他人的库贡献代码(需要对方同意).

自定义Git

详情请见教程原文廖雪峰的Git教程·自定义Git

附:更换远程库流程及遇到的问题

注意:本部分并非廖雪峰的Git教程原文内容,而是我在学习&测试过程中遇到的实际问题,现记录下来作为备用并与大家分享.

我刚开始测试时使用的远程库是Gitlab,在最后准备发布这篇文章时,准备更换到Github上.以下为更换的流程及遇到的一些问题:

  1. 取消本地库与原远程库的关联:

    git remote rm origin

  2. 关联新远程库

    git remote add origin git@github.com:cjlaaa/testGit

  3. 尝试合并远程分支到本地

    git pull

  4. 当然被提示”no tracking information”错误,因为还没有关联本地分支与远程分支.这时候我使用了廖雪峰的Git教程提到的关联命令:

    git branch --set-upstream master origin/master

    提示:

    The --set-upstream flag is deprecated and will be removed. Consider using --track or --set-upstream-to

    看来这个命令已经有点过时了,系统提示要使用新的命令格式:

    git branch --set-upstream-to=origin/master master

  5. 再次使用git pull或git push origin master命令就都正常了.

至此更换新远程库成功.
